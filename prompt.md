# MBTI 성격 유형 테스트 웹 앱 개발 프롬프트 기록

## 1. 시작 화면 요청
MBTI 성격 유형 테스트 웹 앱을 만들거야.
먼저 시작 화면ㅇ르 만들어줘,
- h1태그로 "나의 진짜 성격은?" 이라는 제목을 보여줘
- 그 아래에는 "8개의 간단한 질문으로 알아보는 나의 MBTI 유형" 이라는 설명을 추가해줘.
- 마지막으로 "테스트가 담긴 큰 버튼을 하나 만들어줘.
- 전체적으로 중앙 정렬하고, 깔끔하고 세련된 느낌으로 디자인해줘

## 2. 파일 저장 요청
그리고 내가 입력한 모든 내용을 prompt.md 파일에 저장해줘

## 3. 질문 페이지 및 로직 요청
"테스트 시작하기" 버튼을 누르면 질문 페이지로 넘어가게 해줘.
질문 페이지의 구조는 다음과 같아:
1, 상단에 " Q1." 과 같이 현재 질문 번호를 표시해줘.
2. 중앙에 질문 내용을 보여줘. 첫번째 질문은" 주말에 약속이 없다면?" 이야.
3. 그 아래에 두 개의 선택지 버튼을 만들어줘. 첫 번째 버튼은 "집에 있는다." 두번째 버튼은 "밖에 나가서 사람들을 만난다"야.
4. 사용자가 두 버튼 중 하나를 클릭하면, 다음 질문으로 넘어가게 해줘. 질문 번호와 내용, 선택지도 다음 질문에 맞게 바뀌어야 해.
5. 총 8개의 질문을 모두 보여주고 나면, 마지막에는 결과 페이지로 넘어가야 해.

## 5. 이전 Implementation Plan 백업 및 자동화 요청

## 6. 질문 데이터 적용 요청
질문은 question.md 파일을 확인하고 앱에 적용해줘.

## 7. 동률 시 우선순위 규칙 요청
그러지말고 점수가 같을 경우 질문이 2개씩 있자나. 그중에 첫번째 답에 대한 결과를 우선시 해줘.

## 8. 응답 속도 기준 동률 처리 우선순위 요청
동일한 점수가 나왔을때 혹시 질문화면에서 빨리 넘어 갈수록 점수를 높게 줌으로써 시간을 적게 들이고 다음 질문으로 넘어갔을때 그 시간을 계산해서 그 시간이 적은 질문을 선택하게 해줄 수 있을가? 이 방법이 앞서 정한 규칙보다 좋을 거 같은데

결과 페이지에 좀더 내용을 확인할 수 있는 링크버튼을 넣어줘. 해당 링크는 해당 MBTI에 대해서 자세한 내용이 들어 있는 링크주소야.

## 10. 질문/옵션 랜덤화 및 캐릭터 이미지 적용 요청
좀더 생동감있는 웹 앱으로 업데이트해줘.
- 각 지표 판별 질문을 2배 더 생성해서 그 중 랜덤하게 질문하도록 변경해줘
- 선택지의 순서도 랜덤하게 순서가 바뀌도록 섞어줘
- 질문의 순서가 지금은 E/I, S/N, F/T, J/P 순서대로인데 이전체질문의 순서도 랜덤하게 섞어줘
- 그리고 최종 결과에 대해 예시 이미지를 넣으면 좋을 거 같아. 각결과에 알맞는 귀여운 캐릭터형 이미지를 생성해서 삽입해줘. 이미지는 유형이름에 맞는 이미지를 생성해줘.
- 반드시 생성한 이미지가 사용되어야 해. 이미지 생성 갯수에 제한이 있는 경우, 한 이미지에 16개의 캐릭터가 있는 4x4임미지를 만든 다음, 다운로드 받아, 그런 다음 이미지를 실제의 16개로 분할해. 그런 다음에 각 결과를 각 파일과 연결해
